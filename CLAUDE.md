## プロジェクトの目的

スポット社労士くん（spot-s.or.jp）の無料相談フォーム4種類を提供する。
WordPress側からiframeで埋め込まれている。**このアプリが落ちる/エラーになる = 本番サイトのフォームが使えなくなる**ことを意味する。

公開URL: https://spot-vercel-forms.vercel.app/sme/{consultation, advisor, payroll, subsidy}

---

## 絶対に守ること（最優先）

1. **フォームを「使えない状態」にする変更は絶対に行わない**
   - デプロイ前に必ずローカルで `npm run build` が通ることを確認する
   - ビルドが通らない変更はpush・デプロイしない

2. **Vercelの環境変数 `RESEND_API_KEY` を削除・変更しない**
   - このキーは `spot-s.or.jp` ドメインの送信権限を持つ共有キー
   - 削除するとメール送信が即時停止する
   - キーをコードやGitにコミットしない（.env.localのみ・.gitignore対象）

3. **Vercelプロジェクトの削除・プロジェクト名変更・ドメイン設定の変更をしない**
   - 既存の本番URL（`spot-vercel-forms.vercel.app`）が変わるとWordPress側のiframeが全滅する

4. **`vercel --prod` を実行する前に必ずローカルビルドを通す**
   ```bash
   npm run build
   ```
   エラーが出た場合は絶対にデプロイしない。修正してから再ビルドする。

5. **GitHubリポジトリのmainブランチへの直接pushは最終確認後のみ**
   - 可能であれば別ブランチで作業し、動作確認後にmainへ反映する

## Resend SDKのエラーハンドリング規約

Resend SDKは送信失敗時にthrowしない。必ず以下のパターンで明示的にチェックする：

```ts
const { error } = await resend.emails.send({...})
if (error) throw new Error(error.message)
```

このチェックを省略すると「送信失敗を成功と誤記録する」事故につながるため、
新しいメール送信処理を追加する際は必ず同じパターンに従うこと。

## 共通仕様（4フォーム共通）

- 確認画面・送信完了画面へ遷移した際は、ページ最上部（ヘッダー位置）へスクロールする
  （`FormWizard.tsx` の `topRef` / `scrollToTop()` を使用）
- 受付完了メッセージの文言：「担当者より3営業日以内にご連絡いたします」
- 入力欄には `autoComplete` 属性を設定し、ブラウザの入力補完を有効にする
- 絵文字・AI特有のアイコン（✅️等）は使用しない
- メインカラーは各フォームの `config.color` を使用し、完了画面のアイコン等もこれに統一する

| フォーム | メインカラー |
|---|---|
| はじめてのご相談 | #14b4c6 |
| 顧問契約について | #00a6e2 |
| 給与計算について | #4f6eea |
| 助成金について | #0054ca |

## フォーム個別仕様

### 顧問契約について（advisor）
- 「現在の状況」の選択肢は2つのみ：
  - 社労士と契約中（乗り換え検討）
  - 社労士なし（新規）
  - ※「まだ検討中」は削除済み・復活させない

### 給与計算について（payroll）
- 「特にお困りの点」のチェックボックス項目は削除済み・復活させない

### 助成金について（subsidy）
- 「関心のある助成金・状況」の選択肢は2つのみ：
  - キャリアアップ助成金
  - 育休・産休に関する助成金
- フォーム冒頭に「助成金に関連するご相談・お申し込みは、提携の社会保険労務士による対応となります」の注記を表示する（`type: 'note'` フィールド）
- 通知メールは info@spot-s.jp と contact@roumu-irm.com の両方に送信する

## デプロイ手順（再確認用）

```bash
cd ~/Desktop/spot-vercel-forms
npm install          # 初回・依存関係更新時のみ
npm run build        # 必ず成功を確認する
git add .
git commit -m "変更内容"
git push
vercel --prod
```

デプロイ後は4ページすべてで以下を確認する：
- フォームが表示される
- 確認画面・送信完了まで遷移できる
- スクロールバーが出ない
- 受付完了画面の色がそのフォームのメインカラーになっている

## 既知の制約

- Claudeはこのプロジェクトを「定期的に自動チェックする」ことはできない
  （会話が行われたときのみ動作するため、毎朝の自動巡回は不可能）
